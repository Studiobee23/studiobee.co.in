Showing work before explaining services bypasses the credibility question. The visitor validates quality in their first 8 seconds — then primed to read how you do it.
Fixed Navigation + Pill CTA

Designing for Constraints
Design for Constraints

When limitations made
the design better.

Four moments during the project where a technical or device constraint forced a design rethink — and in each case, the constraint produced a more considered result.

Animation Audit
The hero had 50 animated elements competing for attention — and was crashing iPhones. We treated animation like typography: only what communicates energy earns its place. We reduced, organised, and only kept motion that reinforced the brand's kinetic character.
Design Decision
A leaner animation system that actually says more. The wave strips were kept because they are the hero — the rest was noise. Every removed animation made the remaining ones more powerful.
Ghost Button Duplication
The ghost button appeared to duplicate when it moved through the wave strip inversion system. Animating the whole button broke the illusion — two versions of the button appeared to exist simultaneously.
Design Decision
We redesigned the interaction: only the arrow icon inside the button moves on hover. This made the interaction feel more intentional — a directed gesture, not a floating shape. The constraint forced a more precise micro-interaction.
Texture Rendering Artefacts
A CSS filter for the grain texture caused the entire page to render on one compositor layer, introducing flicker and colour artefacts on mobile — especially on Android Chrome.
Design Decision
A leaner animation system that actually says more. The wave strips were kept because they are the hero — the rest was noise. Every removed animation made the remaining ones more powerful.
Blurred Badge Components
Blurred glass badges on 30+ work cards created a heavy GPU load on lower-performance devices — and visually looked cluttered, blurring both the badge content and the image beneath it.
Design Decision
Replaced backdrop-filter blur with a solid rgba() overlay. The result was actually cleaner and more legible — a reminder that constraints often push toward simplicity. The card labels became sharper, faster, and better.

Case Studies Page Design
Case Studies Page Design

Nine projects, one page,
no uniform grid.

The case-studies.html page was designed separately from the main site. The challenge: display 9 projects in a way that felt curated, not templated. A uniform grid would have looked like a stock template — every project treated with identical weight, which no curated portfolio should do.

A — Uniform 3×3
All same size — feels like a template, not a portfolio
B — Full masonry
Organic feel but no system — random, not curated
C — Mixed portrait+landscape
Some tall, some wide, no system — hard to scan
D — Bento asymmetric (Final)
Each card has a role — portrait, wide, or square
Card Roles
Every card has a job, not just a slot
Portrait cards (9:16) for hero case studies — scrollable like Instagram, dominant in the grid. Wide cards (2-col span) for work with strong horizontal compositions. Square cards for logo/brand work that needs equal framing on both axes.
Interaction Decisions
Hover on desktop, tap-to-reveal on mobile
Hover: card scales 1.02x + colour overlay fades in. No hover text overlay on touch — tapping once reveals the project info panel, tapping again navigates to the case study. Keeps the grid clean on desktop, functional on mobile without hover states.
